## Summary

Map pool rework per the user's curation call: Bone Maze stays, Simple
Tissue and Vascular Network are deleted as the weakest maps, and a
brand-new **Heart Chambers** replaces them — quality over count (the
old "at least 3 maps" target is formally dropped as requirements
Revision 3). The pool is now two designed maps, both with hazards,
chokepoints, and terrain that matters.

## Added — maps/heart_chambers.json (60×60)

Four muscle chambers around a contested central chamber, separated by
bone septa:

- **One-way clockwise bloodstream circuit** through all four outer
  valves — with the flow it's the fast lane around the heart, against
  it brutally slow (verified: 3-cell hop costs 3 with the flow vs 9
  against). Committing to the circuit is a real decision.
- **Valve chokepoints**: outer valves where the circuit crosses the
  septa, plus one narrow inner valve per septum arm into the central
  chamber — natural wall/blocker territory.
- **Central prize**: a habitas point plus the richest AZN node (40)
  inside the central chamber, guarded by a patrol orbiting its walls;
  a second, faster patrol rides the circuit as the fast lane's toll.
- **Terrain that matters**: medium muscle bands along the septa,
  high-density cores padding each chamber's habitas.
- Generated by script with hard gates — the file is only written if
  `validate()` is clean, every habitas/AZN/zone/waypoint cell is
  passable, every objective is pathfinder-reachable from **both**
  spawns, the circuit is measurably directional, and all 3600 cells +
  hazards survive a save/load round-trip. All gates passed first try.

## Removed

- `maps/simple_tissue.json` — the v0.0.1 bring-up map: uniform, no
  hazards, no chokepoints; predates every mechanic that makes the game
  a game.
- `maps/vascular_network.json` — structure was accidental rather than
  designed; its main legacy is the bugs it exposed (v0.0.10/v0.0.12
  spawn fallback, example_strategy_v2's boxed-in fix). Those code
  comments stay, annotated "(shipped until v0.0.16)".

## Changed — references

## Verification

```
$ pytest tests/                    -> 321 passed
Generator gates                    -> all pass (map written first try)
56-match round-robin (8 strategies x 2 maps):
  combat 14W (1480 pts) · explorer 11W (2490) · defense 9W (2210)
  ip_creator 7W (2290) · strategy_v2 7W (2220) · full_roster 5W (1585)
  container 3W (1020) · starter 0W (145)
```

- **Zero-score invariant holds** (v0.0.10): 15 zero-score cases in 56
  matches, every one a needle-kill siege — no strategy is zeroed by
  the map itself.
- **Every strategy scores on both maps individually** — including the
  starter. Notably `example_container`'s relay economy jumps from 180
  pts (bone_maze) to 840 (heart_chambers): the circuit rewards its
  container-ferry pattern; the new map broadens which strategies work.
- **Hazards bite without slaughtering**: 487 bites/28 matches on Bone
  Maze vs 1400 on Heart Chambers (~50/match — the circuit patrol earns
  its toll), yet per-map scores on Heart Chambers are equal or higher
  for 6 of 8 strategies.
- **Showcase** (explorer vs defense, seed 7): full 1500 turns, 220–200
  — the default Run Match pairing stays competitive on the new map.
- **Screenshots inspected**: viewer mid-match on Heart Chambers
  (chambers, circuit arrows, both patrols, 140–130 at T604) and the
  editor with the map loaded (patrol routes, zones, filename shown).
