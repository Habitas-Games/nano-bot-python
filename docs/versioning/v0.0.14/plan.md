# v0.0.14 — M6/M7 Implementation Plan

**Status:** Complete
**Depends on:** [analysis.md](analysis.md)

---

## Approach

Engine first (rules with tests), then the strategies that exercise the
rules, then the UI that shows them, then content (maps), then docs —
each layer verified before the next depends on it. Balance changes are
never eyeballed: every tuning decision cites a replay trace or a full
tournament run.

## Order

1. **Data model**: `MapData.hazards`, loader parse/serialize (hazards
   round-trip even though the editor can't author them — the
   partial-snapshot data-loss trap from v0.0.2 applies), editor
   snapshot/restore, `record_frame(hazards=...)`, `NanoBotData.scan`.
2. **Engine rules** (`simulation_core.py`): `SCAN_FLOOR`, fog-aware
   `_build_map_info` (MapInfo.build signature reworked to take
   precomputed visible lists), `_line_blocked` + LOS in
   `_resolve_attacks` (+ shooting hazards), `habitas_occupied` build
   rejection, `bot_destroyed`/`hazard_destroyed`/`attack_blocked`/
   `hazard_attack` events, `_advance_hazards` phase between movement
   and strategies.
3. **Tests**: 18 new (fog ×4, LOS ×5 incl. the ringed-needle acceptance
   test, exclusivity ×2, hazards ×6, hazard map round-trip ×1) — 319
   total.
4. **Strategies for fog**: combat gains an Explorer spotter + shadowing
   fighter; full_roster's fighter guards the needle when blind;
   defense rebuilt around the watchtower + reactive-wall pattern (three
   traced iterations — see analysis.md §3) with a war-chest cap found
   via a full tournament regression.
5. **Viewer**: fx frame loading, per-frame event overlays + tracers,
   procedural white-cell rendering, replay indexing (id→owner/type,
   notable-event timeline), Events ticker + hazard legend row,
   Replays… picker, seed display/lock, random seeds in both GUI entry
   points.
6. **Menu art**: background with baked title; buttons re-laid to the
   lower third after the first screenshot showed overlap; separate logo
   dropped as redundant.
7. **Editor**: Zone Owner toggle (sidebar group, wired through
   `editor.zone_player` to the zone tool and status text), hazard
   path/blob rendering in the canvas renderer.
8. **Maps**: patrols added to Vascular Network (waypoints
   passability-checked); Bone Maze generated by script with
   pathfinder-verified reachability of every objective from both
   spawns; Simple Tissue left as the hazard-free beginner map.
9. **Docs**: requirements statuses flipped with evidence, guide gains
   fog/white-cell/LOS/exclusivity/seed/replay content and updated bot
   cards, index.html feature cards refreshed, README count.
10. **Verification**: full pytest + editor integration + full app flow;
    84-match tournament ×2 (before/after the war-chest fix); zero-score
    scan; screenshots of menu, editor, and viewer (hazard frame and
    blocked-shot frame).

## Explicit non-goals

- Editor hazard *authoring* tool (MAP-08 stays 🟡) — hazards are
  JSON-authored, but now visible in the editor and safe through save.
- TRN-05 top-3 summary view and the SCO-03 decision — still open in M7.
- Softening `example_combat`'s 21-0 head-to-head record further: the
  design goal was counterplay and survivable losses (victims now score;
  sieges take 1000+ turns against a defender), not making aggression
  unviable.
